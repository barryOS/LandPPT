# 任务计划 - 门窗业务App与Web原型设计

## 1. 任务目标
**一句话描述**: 调研门窗业务流程，并基于调研结果构建Web官网和App的交互原型（HTML版）。

**详细说明**:
- 现状: 用户需要从零开始构建门窗业务的数字化平台。
- 目标: 提供一套涵盖核心业务流程（展示、选配、测量、报价）的高保真HTML原型。
- 收益: 帮助用户明确需求，为后续UI设计和开发提供可视化依据。

## 2. 背景与动机
- 痛点: 门窗业务非标品多、流程长（测量-设计-生产-安装），传统模式效率低，用户体验差。
- 价值: 通过数字化手段（在线选配、预约测量、进度追踪）提升转化率和客户满意度。

## 3. 业务调研与技术方案

### 3.1 门窗业务核心流程
1. **线索获取**: 官网/App展示案例、产品。
2. **需求沟通**: 在线咨询、初步选型。
3. **上门服务**: 预约量尺。
4. **方案报价**: 根据尺寸和选配生成方案与报价。
5. **签约生产**: 确认合同，工厂生产。
6. **安装交付**: 预约安装，验收。
7. **售后服务**: 质保、维修。

### 3.2 功能模块设计
| 模块 | Web端功能 | App端功能 |
|------|-----------|-----------|
| **首页** | 品牌形象、核心卖点、案例轮播 | 快捷入口（预约、进度）、活动推荐 |
| **产品中心** | 详细分类、参数筛选、3D展示 | 列表展示、AR试装（概念） |
| **定制/选配** | 基础选配器（颜色/玻璃/五金） | 简易选配、上传户型图 |
| **服务流程** | 预约表单、经销商查询 | 一键预约、LBS派单 |
| **个人中心** | 订单进度、收藏夹 | 订单全链路追踪、消息通知 |

### 3.3 技术方案
- **原型形式**: HTML5 + Tailwind CSS (CDN)，单页应用模拟。
- **目录结构**:
  - `prototypes/web/`: 官网原型
  - `prototypes/app/`: App原型（移动端尺寸模拟）

## 4. 实施步骤
- Day 1 (Phase 1): 业务调研总结 + 关键页面结构设计 + 基础HTML框架搭建。
- Day 1 (Phase 2): Web端核心页面（首页、产品详情、预约）实现。
- Day 1 (Phase 3): App端核心页面（首页、选配、个人中心）实现。

## 5. 风险与对策
- **风险**: 原型交互不够直观。
  - **对策**: 使用CSS动画和简单的JS交互模拟真实体验。
- **风险**: 业务细节覆盖不全。
  - **对策**: 聚焦核心主流程（Happy Path），边缘情况在文档中备注。

## 6. 成功标准
- [ ] 产出不少于 6 个关键页面的 HTML 原型。
- [ ] 页面包含核心业务逻辑（展示->选配->预约）。
- [ ] 界面美观、布局合理，符合行业审美（简约、高端）。
