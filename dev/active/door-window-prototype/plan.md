# 任务计划 - 门窗业务 App & Web 原型设计

## 1. 任务目标
**一句话描述**: 针对门窗行业"非标定制+重服务"的特性，设计一套包含品牌官网(Web)和业务工具(App)的数字化解决方案原型。

**详细说明**:
- **现状**: 传统门窗生意依赖线下展厅，报价复杂，客户无法直观看到效果，订单跟踪困难。
- **目标**: 
    - **Web端**: 侧重品牌展示、案例引流、简易选品与留资（线索获取）。
    - **App端**: 侧重销售/测量/安装工具、订单全流程跟踪、售后服务（业务交付）。
- **收益**: 提升销售转化率，降低沟通成本，规范交付流程。

## 2. 行业背景与业务调研 (Research)

### 2.1 核心痛点
1.  **非标定制难报价**: 窗户面积、开启方式（平开/推拉/内倒）、型材颜色、玻璃层数（双玻/三玻）、五金配件组合极多，普通用户看不懂，销售算价慢且易错。
2.  **效果不可视**: 用户很难想象这款窗户装在自己家是什么样子。
3.  **交付周期长**: 测量 -> 生产 -> 运输 -> 安装，中间任何环节信息不透明都会导致客诉。
4.  **售后依赖人**: 窗户漏水、五金松动，找不到原来的安装师傅。

### 2.2 竞品/标杆功能分析
| 模块 | 核心功能点 | 价值 |
|------|------------|------|
| **3D选配** | 酷家乐/三维家模式，在线更换型材颜色、玻璃样式 | 解决"所见即所得" |
| **智能算价** | 输入长宽，选择开启扇数量，自动出大概预算 | 解决"价格不透明" |
| **案例库** | 按小区/户型查看已安装案例 | 建立信任，参考决策 |
| **施工交付** | 节点推送（已下单、生产中、发货、预约安装） | 缓解等待焦虑 |

## 3. 技术方案 (原型实现)
我们将使用 **HTML + Tailwind CSS + Alpine.js** 编写高保真、可交互的单页应用(SPA)作为原型，这比静态图片更直观，且代码可复用。

### 架构设计
- **Web端原型**: 响应式设计，适配桌面和手机浏览器。
- **App端原型**: 模拟移动端原生 UI 布局。
